# 클래스 파일 심층 분석 프롬프트

## 사용법
`@[분석할 파일 경로]` 뒤에 아래 프롬프트를 붙여넣으세요.

---

## 프롬프트

```
@[분석할 파일 경로] 이 파일을 아래 기준에 따라 완벽히 분석해줘.

## 1. 코드 주석 달기
- 파일의 모든 코드에 한국어 주석을 달아줘.
- import문, 어노테이션, 필드, 메서드, 내부 로직 각각에 "왜 이것이 필요한지"를 설명해줘.
- 초보 개발자도 이해할 수 있는 수준으로 작성해줘.

## 2. 파일 역할 요약
- 이 파일이 프로젝트 전체에서 어떤 역할을 하는지 한 문단으로 요약해줘.
- 이 파일이 속한 레이어(Controller / Service / Repository / DTO / Config / Test 등)를 명시해줘.

## 3. 연관 파일 매핑
- 이 파일이 직접 의존하는 클래스/인터페이스 목록을 정리해줘.
- 반대로 이 파일을 사용하는 쪽(호출하는 쪽)도 프로젝트 내에서 찾아서 알려줘.
- 관계를 표 형태로 정리해줘:
  | 방향 | 파일명 | 관계 설명 |
  |------|--------|-----------|

## 4. 핵심 로직 흐름
- 주요 메서드별로 실행 흐름을 단계별(1→2→3)로 설명해줘.
- 외부 API 호출, DB 접근, 예외 처리 등 중요 포인트를 강조해줘.

## 5. 사용된 기술/패턴
- 이 파일에서 사용된 디자인 패턴, 프레임워크 기능, 어노테이션의 의미를 설명해줘.
- (테스트 파일인 경우) 테스트 전략(단위/통합), Mock 사용 여부, 테스트 커버리지 범위를 분석해줘.

## 6. 개선 포인트 (선택)
- 코드 품질, 가독성, 테스트 커버리지 측면에서 개선할 점이 있다면 제안해줘.
```

---

## 상황별 커스텀 팁

| 분석 대상 | 추가로 넣으면 좋은 항목 |
|-----------|------------------------|
| **Service** 클래스 | "트랜잭션 경계와 비즈니스 규칙을 중심으로 설명해줘" |
| **Controller** 클래스 | "API 엔드포인트 목록과 요청/응답 스펙을 표로 정리해줘" |
| **Repository** 클래스 | "쿼리 메서드와 실제 생성되는 SQL을 설명해줘" |
| **Config** 클래스 | "설정값의 출처(yml/env)와 영향 범위를 설명해줘" |
| **Test** 클래스 | "어떤 시나리오를 검증하는지, 빠진 테스트 케이스는 없는지 분석해줘" |
| **DTO/Entity** | "필드별 의미와 DB 매핑, Validation 규칙을 설명해줘" |
