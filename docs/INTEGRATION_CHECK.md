# 백엔드 · 프론트엔드 연동 확인

백엔드를 켠 뒤 프론트엔드와 연동되는지 **로그인·회원가입부터** 순서대로 확인하는 방법입니다.

---

## 1. 사전 준비

- **MySQL** 실행 중 (로컬 또는 Docker)
- **DB** `dn_platform` 및 테이블 생성 완료 (`scripts/setup-db.sh` 또는 JPA 자동 생성)
- **backend/.env** 에 `DB_PASSWORD`, (선택) `DATA_API_KEY` 설정

---

## 2. 백엔드 실행

**터미널 1** – 프로젝트 루트(`DN_project01`)에서:

```bash
./scripts/run-backend-dev.sh
```

- `backend/.env` 가 로드되어 DB·공공 API 키가 적용됩니다.
- 콘솔에 `Started DnPlatformApplication` 이 보이고, `http://localhost:8080` 이 떴으면 성공입니다.
- Swagger: http://localhost:8080/swagger-ui.html

---

## 3. 프론트엔드 실행

**터미널 2** – 프론트엔드 디렉터리에서:

```bash
cd frontend
npm install   # 최초 1회
npm run dev
```

- `frontend/.env` 의 `VITE_API_BASE_URL=http://localhost:8080/api` 로 백엔드와 통신합니다.
- **기본 포트 5173** 에서 실행됩니다. (`vite.config.ts` 에 `port: 5173` 설정됨.)
- **이미 5173에서 프론트가 실행 중이면** 그대로 사용하면 됩니다. 백엔드 CORS도 `http://localhost:5173` 으로 허용되어 있어 포트를 맞출 필요 없습니다.
- 브라우저에서 http://localhost:5173 으로 접속합니다.

---

## 4. 연동 확인 순서 (로그인·회원가입부터)

### 4-1. 회원가입

1. 프론트에서 **회원가입** 페이지로 이동 (라우트: `/signup` 또는 로그인 화면의 회원가입 링크).
2. 이메일, 비밀번호, 이름 등 입력 후 가입.
3. **확인**: 가입 완료 후 로그인 페이지로 이동하거나, “가입 완료” 메시지가 나오면 **프론트 → 백엔드 연동 정상**.

- 실패 시: 브라우저 개발자 도구(F12) → Network 탭에서 `POST …/api/auth/signup` 요청/응답 확인.  
  CORS/네트워크 오류가 있으면 백엔드 `application.yml` 의 `cors.allowed-origins` 에 `http://localhost:5173` 이 포함되어 있는지 확인.

### 4-2. 로그인

1. **로그인** 페이지에서 방금 가입한 계정으로 로그인.
2. **확인**: 로그인 성공 후 메인/마이페이지로 이동하고, 상단에 로그인된 사용자 정보가 보이면 **로그인 API 연동 정상**.

- 실패 시: Network 탭에서 `POST …/api/auth/login` 응답 확인.  
  401이면 이메일/비밀번호 확인, 404/500이면 백엔드 로그 확인.

### 4-3. 마이페이지(내 정보)

1. 로그인된 상태에서 **마이페이지**(또는 “내 정보”) 로 이동.
2. **확인**: `GET …/api/auth/me` 가 성공해 이메일·이름 등이 표시되면 **인증·토큰 연동 정상**.

### 4-4. 동물 목록

1. **유기동물 목록** 페이지로 이동.
2. **확인**: `GET …/api/animals` 가 호출되고, 목록(또는 빈 목록)이 표시되면 **동물 API 연동 정상**.  
   (공공 API 동기화를 했다면 실제 데이터가 보일 수 있습니다.)

### 4-5. (선택) 공공데이터 동기화 후 동물 데이터

- 관리자 계정으로 **공공데이터 동기화**를 한 뒤, 다시 동물 목록을 새로고침해 데이터가 채워지는지 확인하면 **백엔드–프론트–공공 API** 까지 한 번에 확인할 수 있습니다.  
  자동 확인 스크립트: `./scripts/verify-public-api.sh` (백엔드 실행 중일 때만).

---

## 5. 요약 체크리스트

| 순서 | 확인 항목           | 성공 기준                          |
|------|----------------------|------------------------------------|
| 1    | 백엔드 실행         | `localhost:8080` 접속 가능         |
| 2    | 프론트 실행         | `localhost:5173` 접속 가능         |
| 3    | 회원가입             | 가입 완료 후 다음 단계로 진행       |
| 4    | 로그인               | 로그인 후 메인/마이페이지 진입     |
| 5    | 마이페이지           | 내 정보(이메일·이름 등) 표시       |
| 6    | 동물 목록            | 목록 조회(데이터 유무와 무관) 성공  |

위 순서대로 확인하면 **백엔드를 켠 상태에서 프론트와 연동 여부를 로그인·회원가입부터** 검증할 수 있습니다.
