# 입양 파트 ↔ 쇼핑몰 구조 매핑

입양(동물 목록·상세·신청)은 **쇼핑몰 구조에서 일정 부분을 차용**합니다.  
백엔드는 **스프링부트_Gemini.md**에 나오는 Spring Boot(쇼핑몰) 아키텍처를 참조하고,  
**프론트엔드는 책과 다르게 React**로 구현합니다.

---

## 1. 참조 문서

| 문서 | 내용 | View/프론트 |
|------|------|-------------|
| **스프링부트_Gemini.md** | Spring Boot 쇼핑몰: Item, ItemImg, Repository, Service, 이미지 업로드, 주문/장바구니 | **Thymeleaf** (책 기준) |
| **DN_project** | 62댕냥이 매칭 플랫폼, 입양 파트 = 쇼핑몰 스타일 차용 | **React** (Vite + TypeScript) |

- **백엔드**: 스프링부트_Gemini.md의 **엔티티 설계, Repository, Service, Controller 패턴, 이미지 업로드, DTO** 등을 참조하여 **Animal / AnimalImage / Adoption** 도메인에 맞게 적용.
- **프론트엔드**: 책의 Thymeleaf 대신 **React**로 동물 목록(쇼핑몰 스타일), 동물 상세, 입양 신청 화면 구현.

---

## 2. 도메인 매핑 (쇼핑몰 → 입양)

| 쇼핑몰 (스프링부트_Gemini.md) | DN 입양 (우리 프로젝트) | 비고 |
|------------------------------|-------------------------|------|
| **Item** (상품) | **Animal** (동물) | 목록/상세의 주체 |
| **ItemImg** (상품 이미지) | **AnimalImage** (동물 이미지) | 1:N, 대표 이미지(repImgYn → isMain) |
| **ItemSellStatus** (SELL/SOLD_OUT) | **AnimalStatus** (PROTECTED/ADOPTED/FOSTERING) | 상태 Enum |
| **Order** (주문) | **Adoption** (입양/임보 신청) | “주문하기” → “입양/임보 신청하기” |
| **OrderItem** (주문 상품) | — | 입양은 1회에 1마리 신청이라 단순화 가능 |
| **Cart** / **CartItem** (장바구니) | — | 입양에서는 사용하지 않음 (선택 사항) |
| **Member** | **User** | 로그인 회원 |
| **FileService** / 이미지 업로드 | **FileService** 또는 동일 패턴 | UUID 파일명, Resource 핸들러 매핑 |

## 3. 기능 매핑 (쇼핑몰 → 입양)

| 쇼핑몰 기능 (문서) | 입양 파트 기능 | 백엔드 참조 | 프론트 (React) |
|--------------------|----------------|-------------|----------------|
| 상품 목록 (페이징, 필터) | 동물 목록 (종, 크기, 지역, 상태 필터 / 정렬) | ItemRepository 쿼리 메소드·Querydsl → AnimalRepository | 목록 페이지, 카드 그리드, 필터/정렬 UI |
| 상품 상세 (이미지 갤러리, 상세 정보) | 동물 상세 (이미지 갤러리, 상세 정보, 보호소 정보) | Item + ItemImg 조회 → Animal + AnimalImage | 상세 페이지, 갤러리, “입양 신청” 버튼 |
| 상품 등록 (폼 + 이미지 multipart) | 동물 수동 등록 (보호소 관리자, 이미지 업로드) | ItemService.saveItem + ItemImgService → AnimalService | 관리자용 등록 폼 (추후) |
| 주문하기 (Order) | 입양/임보 신청하기 (Adoption) | OrderService.order → AdoptionService | 신청 폼, 내 신청 목록 |
| 장바구니 담기/주문 | — | 사용 안 함 | — |

## 4. 입양 화면 구조 (쇼핑몰 스타일 + React)

구조 문서(DN_project_structure_260128.md) 4.2절 기준:

- **동물 목록**: 필터(종, 크기, 지역, 상태) + 정렬(최신순, 인기순) + **썸네일 카드 그리드** (이름, 종, 나이, 크기, 상태, 보호소, [상세보기]).
- **동물 상세**: 이미지 갤러리, 상세 정보(품종, 성별, 중성화, 예방접종, 성격 등), 보호소 정보, **“입양 신청하기” / “임시보호 신청하기”** 버튼.

백엔드 API(목록/상세/신청)는 스프링부트_Gemini.md의 **Controller·Service·Repository·DTO 패턴**을 참조하고,  
프론트는 **React**로 위 화면을 구현합니다 (쇼핑몰 UI를 “동물” 도메인에 맞게 적용).

## 5. 요약

| 구분 | 쇼핑몰 (책/문서) | DN 입양 |
|------|------------------|--------|
| **백엔드** | Spring Boot, JPA, Thymeleaf | Spring Boot, JPA, **REST API** (문서 구조 참조) |
| **프론트** | Thymeleaf + Bootstrap | **React (Vite + TypeScript)** |
| **목록/상세** | 상품 목록·상세 | 동물 목록·상세 (같은 UX 패턴) |
| **주문** | Order / OrderItem | Adoption (입양/임보 신청) |

즉, **입양 부분은 쇼핑몰 구조에서 많이 빌려오고, 백엔드는 스프링부트_Gemini.md를 참조하며, 프론트만 책과 다르게 React로 간다**는 점을 이 문서로 정리합니다.
