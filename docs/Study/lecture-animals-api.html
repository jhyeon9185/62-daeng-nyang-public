🔑 핵심 포인트

  • 공공 API ↔ 우리 서버: 사용자 요청 시마다 공공 API를 호출하지 않음. 주기적 동기화(Sync)로 우리 DB에 저장 후 서빙.
  • 프론트엔드 ↔ 백엔드: 프론트는 우리 백엔드 REST API만 호출. 공공 API를 직접 호출하지 않음.
  • Kakao Map: 프론트엔드에서 SDK를 로드하여 브라우저에서 직접 렌더링 (백엔드 무관).

3. Entity — 데이터 모델

3-1. Animal 엔티티

파일: domain/Animal.java — DB animals 테이블에 매핑

@Entity
@Table(name = "animals", indexes = {
    @Index(name = "idx_animals_species", columnList = "species"),
    @Index(name = "idx_animals_status", columnList = "status"),
    @Index(name = "idx_animals_shelter", columnList = "shelter_id"),
    @Index(name = "idx_animals_public_api_id", columnList = "public_api_animal_id")
})
public class Animal {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @ManyToOne(fetch = FetchType.LAZY)         // Shelter와 N:1 관계
    @JoinColumn(name = "shelter_id", nullable = false)
    private Shelter shelter;

    private String publicApiAnimalId;           // 공공 API 유기번호 (식별 키)
    @Enumerated(EnumType.STRING)
    private Species species;                    // DOG, CAT
    private String breed, name;
    private Integer age;
    @Enumerated(EnumType.STRING)
    private Gender gender;                      // MALE, FEMALE
    @Enumerated(EnumType.STRING)
    private Size size;                          // SMALL, MEDIUM, LARGE
    private BigDecimal weight;
    @Column(columnDefinition = "TEXT")
    private String description;
    private Boolean neutered, vaccinated;
    private String imageUrl;
    @Enumerated(EnumType.STRING)
    private AnimalStatus status;                // PROTECTED, ADOPTED, FOSTERING

    private LocalDate registerDate;
    private Instant createdAt, updatedAt;

    @PrePersist void prePersist() { /* 생성시각 자동 */}
    @PreUpdate  void preUpdate()  { /* 수정시각 자동 */}
}

📘 학습 포인트

  • @ManyToOne(fetch = FetchType.LAZY): 보호소와의 연관관계. LAZY → 실제 사용 시 로딩.
  • @Enumerated(EnumType.STRING): Enum을 문자열 형태로 DB에 저장 (DOG, CAT 등).
  • @PrePersist / @PreUpdate: JPA 라이프사이클 콜백으로 생성/수정 시각 자동 관리.
  • @Index: 인덱스로 조회 성능 최적화. publicApiAnimalId는 공공 API와 연결하는 핵심 키.

📘 학습 포인트 — WebClient vs RestTemplate

  • WebClient: Spring 5+ 권장 HTTP 클라이언트. 비동기 지원 (.block()으로 동기 전환).
  • UriComponentsBuilder: URL 인코딩을 안전하게 처리하며 쿼리 파라미터 조립.
  • @Value("${public-api.service-key:}"): application.properties/yml에서 API 키를 주입.
  • 응답이 단일 객체일 수도, 배열일 수도 있어서 파싱 로직에서 분기 처리.

4-2. AnimalSyncService — 데이터 동기화

파일: service/AnimalSyncService.java (약 500줄)

동기화 전략: 증분 동기화 (Incremental Sync)

  1. 기간 지정 조회: 최근 N일 내 변경된 동물만 API로 요청 (매일 전체 조회 X)
  2. Upsert 패턴: publicApiAnimalId로 DB 조회 → 있으면 UPDATE, 없으면 INSERT
  3. 상태 기반 삭제: 입양/안락사/자연사/반환 → DB에서 삭제 (보호중/임보만 유지)
// 핵심 메서드: upsertAnimal
private SyncAction upsertAnimal(AnimalItem item) {
    AnimalStatus status = mapStatus(item.getProcessState());
    var existing = animalRepository.findByPublicApiAnimalId(item.getDesertionNo());

    // 보호 대상이 아닌 동물은 저장하지 않고, 기존에 있으면 삭제
    if (status == null) {
        if (existing.isPresent()) {
            animalRepository.delete(existing.get());
            return SyncAction.REMOVED;
        }
        return SyncAction.SKIPPED;
    }
    // 기존 데이터가 있으면 UPDATE, 없으면 CREATE
    if (existing.isPresent()) {
        updateAnimalFromApi(existing.get(), item);
        return SyncAction.UPDATED;
    }
    createAnimalFromApi(item);
    return SyncAction.ADDED;
}

11-1. KakaoMap.tsx — 지도 컴포넌트

동작 원리

  1. loadKakaoScript(): <script> 태그를 동적 생성하여 Kakao Maps SDK 로드.
  2. SDK URL: //dapi.kakao.com/v2/maps/sdk.js?appkey=KEY&libraries=services&autoload=false
  3. autoload=false로 수동 로딩 → kakao.maps.load()로 초기화.
  4. 위·경도가 있으면 바로 지도 표시, 없으면 Geocoder.addressSearch()로 주소→좌표 변환.
// 핵심: 좌표가 있으면 바로, 없으면 주소 검색
if (hasCoords) {
  showMap(latitude!, longitude!);  // 바로 표시
} else {
  geocoder.addressSearch(address, (result, status) => {
    if (status === kakao.maps.services.Status.OK && result[0]) {
      showMap(parseFloat(result[0].y), parseFloat(result[0].x));
    }
  });
}

const showMap = (lat: number, lng: number) => {

❓ 강의 중 Q&A 예상 질문

  1. Q: 왜 공공 API를 직접 호출하지 않나요?
    A: 느린 응답속도, 호출 제한, 데이터 가공 필요 → 동기화 후 우리 DB에서 서빙이 효율적.
  2. Q: 동기화는 언제 실행되나요?
    A: syncDailySchedule()은 스케줄러로 매일 자동 실행. 관리자가 수동 실행도 가능.
  3. Q: 디바운스를 왜 사용하나요?
    A: 키 입력마다 API 호출하면 서버 부하 발생. 입력이 멈춘 후 한 번만 호출.
  4. Q: Entity와 Response DTO를 분리하는 이유?
    A: Entity 직접 노출 시 내부 구조 노출, 순환 참조, 불필요한 필드 전송 문제. DTO로 필요한 것만 전달.
  5. Q: KakaoMap에서 위·경도가 없으면?
    A: Geocoder를 사용해 주소 문자열을 좌표로 변환해서 표시합니다.

package com.dnproject.platform.service;

import com.dnproject.platform.dto.publicapi.AnimalItem;
import com.fasterxml.jackson.databind.ObjectMapper;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Service;
import org.springframework.web.reactive.function.client.WebClient;
import org.springframework.web.util.UriComponentsBuilder;

@Slf4j
@Service
public class PublicApiService {

    private final WebClient.Builder webClientBuilder;
    private final ObjectMapper objectMapper;

    // application.yml에서 API 키 주입
    @Value("${public-api.service-key}")
    private String serviceKey;

    @Value("${public-api.base-url}")
    private String baseUrl;

    // 유기동물 목록 조회 — 핵심 메서드
    public List<AnimalItem> getAbandonedAnimals(
            String uprCd, String orgCd, String upkind,
            String state, LocalDate bgnde, LocalDate endde,
            int pageNo, int numOfRows) {

        // API 키 미설정 시 안전하게 빈 리스트 반환
        if (serviceKey == null || serviceKey.isBlank()) {
            log.warn("공공 API 서비스 키가 설정되지 않았습니다.");
            return Collections.emptyList();
        }

        // 1단계: UriComponentsBuilder로 쿼리 파라미터 안전하게 조립
        var builder = UriComponentsBuilder
            .fromHttpUrl(baseUrl + "/abandonmentPublic")
            .queryParam("serviceKey", serviceKey)
            .queryParam("pageNo", pageNo)
            .queryParam("numOfRows", numOfRows)
            .queryParam("_type", "json"); // JSON 응답 요청

        // 조건부 파라미터 추가 (null이 아닌 것만)
        if (uprCd != null) builder.queryParam("upr_cd", uprCd);
        if (upkind != null) builder.queryParam("upkind", upkind);

        // 2단계: WebClient로 GET 요청 → .block()으로 동기 호출
        String json = webClientBuilder.build()
            .get().uri(builder.build().toUriString())
            .retrieve()
            .bodyToMono(String.class)
            .block();

        // 3단계: JSON 파싱 → AnimalItem 리스트 변환
        return parseAnimalItems(json);
    }

    // JSON 응답에서 item 배열 추출
    private List<AnimalItem> parseAnimalItems(String json) {
        var root = objectMapper.readTree(json);
        var items = root.path("response").path("body")
                       .path("items").path("item");
        return objectMapper.readerForListOf(AnimalItem.class)
                          .readValue(items);
    }
}

package com.dnproject.platform.service;

import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.scheduling.annotation.Scheduled;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Slf4j
@Service
@RequiredArgsConstructor
public class AnimalSyncService {

    private final PublicApiService publicApiService;
    private final AnimalRepository animalRepository;

    // 매일 오전 3시 자동 동기화 (@Scheduled 어노테이션)
    @Scheduled(cron = "0 0 3 * * *")
    @Transactional
    public void syncDailySchedule() {
        log.info("정기 데이터 동기화 시작...");
        syncFromPublicApi(7, null, null);
    }

    // 핵심: Upsert 로직
    // publicApiAnimalId(유기번호)로 기존 데이터 검색 →
    // 있으면 UPDATE, 없으면 INSERT, 비보호 상태면 DELETE
    private SyncAction upsertAnimal(AnimalItem item) {
        AnimalStatus status = mapStatus(item.getProcessState());
        var existing = animalRepository
            .findByPublicApiAnimalId(item.getDesertionNo());

        // 비보호 동물(입양/안락사/반환) → DB에서 삭제
        if (status == null) {
            if (existing.isPresent()) {
                animalRepository.delete(existing.get());
                return SyncAction.REMOVED;
            }
            return SyncAction.SKIPPED;
        }

        // 기존 데이터 있으면 UPDATE
        if (existing.isPresent()) {
            updateAnimalFromApi(existing.get(), item);
            return SyncAction.UPDATED;
        }

        // 신규 데이터 → INSERT
        createAnimalFromApi(item);
        return SyncAction.ADDED;
    }

    // 공공API 상태 → 우리 Enum으로 변환
    private AnimalStatus mapStatus(String processState) {
        if (processState == null) return null;
        if (processState.contains("보호중") ||
            processState.contains("공고중"))
            return AnimalStatus.PROTECTED;
        if (processState.contains("임시보호"))
            return AnimalStatus.FOSTERING;
        return null; // 입양/안락사/반환 등 → 삭제 대상
    }

    // 동기화 결과를 불변 record로 반환
    public record SyncResult(
        int addedCount, int updatedCount, int removedCount
    ) {}
}

// 주소 또는 좌표로 보호소 위치를 지도에 표시
// ============================================
import { useEffect, useRef } from 'react';

// 카카오 SDK 전역 타입 선언
declare global {
  interface Window { kakao: any; }
}

interface Props {
  address?: string;
  latitude?: number;
  longitude?: number;
  height?: number;
}

export default function KakaoMap(props: Props) {
  const containerRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    // 1단계: SDK 스크립트 동적 로드 (중복 방지)
    const script = document.createElement('script');
    script.src = `//dapi.kakao.com/v2/maps/sdk.js`
      + `?appkey=${import.meta.env.VITE_MAP_API_KEY}`
      + `&libraries=services&autoload=false`;
    document.head.appendChild(script);

    // 2단계: SDK 로드 완료 후 지도 초기화
    script.onload = () => {
      window.kakao.maps.load(() => {
        const hasCoords = props.latitude != null;

        if (hasCoords) {
          // 좌표가 있으면 바로 지도 표시
          showMap(props.latitude!, props.longitude!);
        } else {
          // 좌표 없으면 Geocoder로 주소→좌표 변환
          const geocoder = new window.kakao.maps
            .services.Geocoder();
          geocoder.addressSearch(props.address,
            (result, status) => {
              if (status === window.kakao.maps
                .services.Status.OK) {
                showMap(
                  parseFloat(result[0].y),
                  parseFloat(result[0].x)
                );
              }
            });
        }
      });
    };

    // 지도 생성 + 마커 표시 헬퍼
    const showMap = (lat: number, lng: number) => {
      const center = new window.kakao.maps.LatLng(lat, lng);
      const map = new window.kakao.maps.Map(
        containerRef.current, { center, level: 3 }
      );
      new window.kakao.maps.Marker({ position: center })
        .setMap(map);
    };
  }, [props.address, props.latitude]);

  return <div ref={containerRef}
    style={{ width: '100%', height: props.height || 400 }}
  />;
}

📋 역할

DB 테이블과 1:1 매핑되는 JPA 엔티티입니다. 유기동물의 모든 속성(종류, 품종, 나이, 상태, 보호소 관계 등)을 정의합니다.

⚙️ 기술 포인트

  • @Entity — JPA가 이 클래스를 DB 테이블로 관리
  • @Enumerated(EnumType.STRING) — enum을 문자열로 저장하여 DB 가독성 확보
  • @ManyToOne(LAZY) — Shelter와 N:1 관계. 실제 사용 시에만 로딩
  • Builder 패턴 — 객체 생성 시 가독성과 안전성 확보
  • BaseTimeEntity 상속 — createdAt, updatedAt 자동 관리
  • @Index — species, status, shelter_id, publicApiAnimalId에 인덱스로 조회 성능 최적화

📋 역할

공공데이터포털의 국가동물보호정보시스템 API와 HTTP 통신하여 유기동물·보호소·품종 데이터를 JSON으로 받아오는 클라이언트 서비스입니다.

🔑 핵심 메서드

  • getAbandonedAnimals() — 시도코드·시군구·축종·상태·날짜 범위를 파라미터로 공공 API 호출. @Value로 주입받은 serviceKey 사용.
  • parseAnimalItems() — JSON 응답에서 response.body.items.item 경로를 탐색하여 AnimalItem 리스트로 변환.
  • getKindList() — 품종코드→품종명 매핑용 API 호출
  • getShelters() — 보호소 정보 조회

⚙️ 기술 포인트

  • WebClient (Spring WebFlux) — 비동기 HTTP 클라이언트이지만 .block()으로 동기 호출
  • UriComponentsBuilder — 쿼리 파라미터를 안전하게 조합
  • ObjectMapper (Jackson) — JSON 파싱에 사용
  • API 키 미설정 시 빈 리스트 반환으로 안전하게 동작

📋 역할

공공 API에서 받아온 AnimalItem을 DB의 Animal 엔티티로 변환하여 증분 동기화(Incremental Sync)를 수행합니다. 핵심은 Upsert 패턴입니다.

🔑 핵심 메서드

  • syncFromPublicApi() — N일치 변경분만 동기화. 개·고양이 각각 페이지네이션으로 전체 조회
  • upsertAnimal() — 유기번호로 기존 DB 검색 → 있으면 UPDATE, 없으면 INSERT, 비보호 상태면 DELETE
  • mapStatus() — 공공 API의 processState를 AnimalStatus enum으로 변환. "보호중/공고중"→PROTECTED, "임시보호"→FOSTERING, 기타→null(삭제)
  • resolveImageUrl() — popfile1→popfile2→popfile 순서로 이미지 URL 우선순위 처리
  • findOrCreateShelter() — 보호소가 DB에 없으면 자동 생성 (주소에서 시도·시군구 파싱)

⚙️ 기술 포인트

  • @Transactional — 동기화 전체가 하나의 트랜잭션으로 처리
  • @Scheduled(cron) — 매일 오전 3시 자동 실행
  • SyncResult record — added/updated/removed 카운트를 불변 객체로 반환
  • ConcurrentHashMap — 품종코드→품종명 캐시 (thread-safe)

📋 역할

/api/animals 경로의 REST API 진입점입니다. HTTP 요청 파라미터를 파싱하여 Service에 위임합니다.

🔑 핵심 엔드포인트

  • GET /api/animals — 필터(종류·상태·크기·지역·검색어) + 정렬(랜덤/기준별) 지원하는 목록 조회
  • GET /api/animals/{id} — 상세 조회
  • GET /api/animals/recommendations — 사용자 선호도 기반 추천 (인증 필요)
  • POST / PUT / DELETE — 관리자의 CRUD 작업

⚙️ 기술 포인트

  • @RestController + @RequestMapping — REST API 매핑
  • ApiResponse<T> — 표준화된 응답 래퍼 (success/message/data)
  • sort 파라미터가 "random"이면 랜덤 쿼리, 아니면 정렬 기준 파싱

📋 역할

/api/admin/animals 경로의 관리자 전용 API입니다. 공공데이터 동기화 트리거와 이력 관리를 담당합니다.

🔑 핵심 엔드포인트

  • POST /sync-from-public-api — 수동 동기화 트리거 (days, maxPages, species 파라미터)
  • DELETE /cleanup-invalid — 동기화 실행 후 비보호 동물 일괄 정리
  • GET /sync-history — 동기화 이력 페이지네이션 조회

⚙️ 기술 포인트

  • SyncTriggerType.MANUAL — 수동 vs 자동 동기화 구분하여 이력 추적
  • 동기화 실패해도 DB 정리는 계속 진행 (부분 실패 허용)

📋 역할

Spring Data JPA의 Repository 인터페이스입니다. 인터페이스만 선언하면 Spring이 자동으로 구현체를 생성합니다.

🔑 핵심 메서드

  • findByPublicApiAnimalId() — 유기번호로 기존 동물 검색 (동기화 시 중복 판별)
  • findAllWithFilters() — @Query JPQL로 동적 필터링
  • findWithFiltersRandom() — FUNCTION('RAND')로 랜덤 정렬

⚙️ 기술 포인트

  • JpaRepository — CRUD + 페이지네이션 자동 제공
  • :param IS NULL OR ... 패턴 — null이면 해당 조건 무시 (동적 필터)
  • JOIN a.shelter s — Shelter 테이블의 지역 정보 활용

📋 역할

동물 목록을 보여주는 React 페이지 컴포넌트입니다. 전체 목록과 선호도 기반 추천 모드를 제공합니다.

🔑 핵심 로직

  • FavoriteButton — 하트 버튼으로 찜하기 (로그인 필요)

⚙️ 기술 포인트

  • React Hooks — useState, useEffect로 상태 관리
  • createPortal — 선호도 모달을 body에 직접 렌더링
  • 서버 사이드 필터링 + 페이지네이션 → 대량 데이터 효율적 처리

📋 역할

동물 상세 정보를 보여주는 React 페이지입니다. 보호소 위치를 카카오 지도로 표시합니다.

🔑 핵심 로직

  • useParams() — URL의 :id 파라미터에서 동물 ID 추출
  • animalApi.getById(id) — 상세 데이터 API 호출
  • <KakaoMap> — 보호소 주소 또는 위경도로 지도 표시
  • 이미지 확대 모달, 전화 문의 문구 복사 기능

📋 역할

카카오 지도 SDK를 래핑한 React 컴포넌트입니다. 주소 또는 좌표로 보호소 위치를 표시합니다.

🔑 핵심 로직

  • loadKakaoScript() — SDK 스크립트 동적 로드 (중복 방지). autoload=false로 수동 초기화.
  • 위경도 우선 — latitude/longitude가 있으면 바로 지도 표시, 없으면 Geocoder로 주소→좌표 변환
  • Geocoder.addressSearch() — 주소 문자열을 위경도로 변환하는 카카오 API

⚙️ 기술 포인트

  • useRef — DOM 컨테이너와 Map 인스턴스 참조 유지

  • import.meta.env.VITE_MAP_API_KEY — Vite 환경변수로 API 키 관리
